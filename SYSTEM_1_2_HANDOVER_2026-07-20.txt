WHAT IS STILL NOT PROVEN / NOT FINISHED
=======================================
1. External proposal connectors (Google Drive, OneDrive, SharePoint, Notion, Confluence, Gmail/Outlook account sync, WhatsApp, Teams, Slack, CRM, ERP, LMS, accounting and ticketing platforms) require provider applications, credentials, consent scopes, and connector-specific live validation. Existing connector interfaces are not equivalent to live integrations.
2. Legacy binary .ppt conversion code requires LibreOffice/soffice on the host; it was not live-proven on this machine. PPTX is proven.
3. MSG bounded attachment extraction has automated coverage, but no representative real-world .msg attachment fixture was supplied for a live Kafka proof.
4. The safe crawler intentionally ingests bounded readable page text. It does not mirror arbitrary video/audio/other-media folders from the universal scraper because that would bypass the knowledge-ingestion size/type and security policy. Linked binary document ingestion can be added as a separately bounded feature.
5. All knowledge categories have end-to-end automated extraction/publish tests, but the live mixed validation document naturally exercised policy, pricing, plan and FAQ rather than every category.
6. MinIO is live locally and uses persistent Docker storage; production still needs a managed S3-compatible endpoint, rotated credentials, bucket lifecycle/retention policy, encryption policy, backups and restore drills.
7. Production load tests, alert dashboards, multi-node Kafka/Qdrant/PostgreSQL/FerretDB high availability, disaster recovery, and long-duration soak tests are not established by this local proof.
8. Actual SDR/sales/customer-success worker behavior belongs to System 5. System 2 now exposes and tests their shared context contract, but it does not implement those workers.
9. Brevo/campaign delivery remains deliberately deferred, as requested.

BOTTOM LINE
===========
The PDF/DOCX/document-and-safe-website knowledge slice is ready for controlled onboarding use: it queues durably, indexes with layout metadata, requires human approval, synchronizes PostgreSQL and Qdrant through a retryable outbox, retrieves approved evidence only, exposes conflicts instead of guessing, cites every knowledge layer, remembers onboarding/conversations, and enforces tenant boundaries.

The full System 1 product proposal is not finished because the many external business-source connectors and production operations listed above are separate integrations. System 2's functional knowledge core is working; its remaining work is primarily production hardening and downstream System 5 worker adoption.
